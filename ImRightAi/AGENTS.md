## 주석 작성 규칙

- 모든 소스 파일은 최상단에 이 파일의 책임을 설명하는 짧은 한국어 주석을 반드시 작성한다.
- public method와 로직이 복잡한 method에는 목적과 핵심 동작을 설명하는 한국어 주석을 작성한다.
- getter/setter, 단순 매핑, 너무 자명한 유틸 메서드에는 주석을 생략한다.
- 주석은 코드가 "무엇을 하는지"를 반복하지 말고, 왜 이 코드가 필요한지, 어떤 제약이 있는지, 어떤 부작용이나 중요한 판단이 있는지를 설명해야 한다.
- 주석은 리뷰어가 빠르게 이해할 수 있도록 짧고 명확하게 작성한다.
- 정책, 제약, 예외 처리, 멱등성, 상태 전이처럼 리뷰에서 중요한 내용은 주석으로 드러낸다.
- 구현이 변경되면 관련 주석도 같은 수정에서 함께 갱신한다.
- 모든 주석은 한국어로 작성한다.

## 주석 작성 방식

- 파일 최상단 주석은 "이 파일이 맡은 책임"만 1~3줄로 설명한다.
- 메서드 주석은 "언제 호출되는지", "무엇을 보장하는지", "주의할 제약이 무엇인지" 위주로 작성한다.
- 코드만 읽어도 바로 보이는 내용은 주석으로 쓰지 않는다.
- 불필요한 장문 설명 대신, 리뷰에 필요한 판단 근거를 우선해서 남긴다.

 * 비즈니스 규칙은 여기서 조합하고, DB 접근은 repository에 위임한다.
 */

 * 오늘 무료 플레이 가능 여부를 먼저 확인하고,

 */

 */
