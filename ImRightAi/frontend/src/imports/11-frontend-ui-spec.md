# ImRight AI
## Frontend UI Spec

> 목적: 프론트 UI 구조와 화면 구성을 정의해 Figma MCP 및 실제 구현의 기준으로 사용한다.
> 상태: Draft v0.1

---

## 1. UI 설계 원칙

- 모바일 우선으로 설계하되 데스크톱에서도 자연스럽게 확장된다.
- 게임 시작까지의 진입 장벽은 최소화한다.
- 결과는 즉시 보여주고, 저장/랭킹/재도전은 결과 이후 선택 행동으로 둔다.
- 하단 탭 구조는 단순하게 유지한다.
- 게임 진행 화면에서는 집중도를 해치지 않도록 탭과 주변 요소를 최소화한다.

### 반응형 원칙

- UI 문서는 이 한 장으로 관리한다.
- 모바일, 태블릿, 데스크톱을 따로 문서로 쪼개지 않고 하나의 규칙 안에서 대응한다.
- 모바일은 1열 중심, 태블릿은 넓은 카드, 데스크톱은 2열 확장을 허용한다.
- 핵심 CTA는 어떤 화면 크기에서도 첫 화면 안에 보이게 한다.

### 기준 프레임

- Mobile S: `360x800`
- Mobile M: `390x844`
- Mobile L: `430x932`
- Tablet: `768x1024`
- Desktop: `1440x1024`

### 브레이크포인트 권장

- `0-767px`: 모바일
- `768-1199px`: 태블릿
- `1200px+`: 데스크톱

## 2. 글로벌 네비게이션

### 하단 탭 4개

1. `홈`
2. `플레이`
3. `랭킹`
4. `내 기록`

### 탭 역할

- `홈`
  - 오늘의 주제 노출
  - 현재 무료 플레이 가능 여부
  - 바로 플레이 진입

- `플레이`
  - 실제 토론 진행
  - 진행 중 게임 복귀

- `랭킹`
  - 일일 리더보드
  - 상위 유저, 내 순위

- `내 기록`
  - 저장한 게임 기록
  - 최고 점수
  - 닉네임 설정/변경

### 탭 표시 원칙

- 일반 화면에서는 하단 탭 표시
- 실제 토론 중에는 탭 숨김 또는 축소
- 결과 화면에서는 탭 복귀 가능
- 데스크톱에서는 하단 탭을 상단 또는 좌측 네비로 치환 가능

## 3. 주요 화면 구조

### 3.1 홈

가장 먼저 보이는 핵심 화면.

#### 목적

- 오늘의 주제를 바로 이해시킨다
- 무료 1회 또는 재도전 가능 상태를 보여준다
- `지금 시작` CTA를 가장 강하게 노출한다

#### 주요 구성

- 상단 로고 또는 서비스 타이틀
- 오늘의 주제 카드
  - 제목
  - 옵션 A
  - 옵션 B
  - 오늘 날짜
- 상태 요약 카드
  - 오늘 무료 플레이 가능 여부
  - retry ticket 보유 수
  - 닉네임 설정 여부
- 메인 CTA
  - `토론 시작하기`
- 보조 CTA
  - `랭킹 보기`
- 하단 배너/앵커 광고 영역

#### 상태

- 첫 무료 플레이 가능
- 무료 소진 + ticket 있음
- 무료 소진 + ticket 없음
- 닉네임 미설정
- 네트워크 오류

---

### 3.2 플레이 진입

홈에서 시작 버튼을 누르면 나오는 준비 화면 또는 바로 진입 플로우.

#### 목적

- 유저가 A/B 중 하나를 선택하게 한다
- 선택 후 즉시 게임을 시작한다

#### 주요 구성

- 주제 헤더
- A/B 선택 카드 2개
- 선택 설명 텍스트
- 시작 버튼

#### UX 원칙

- 선택은 한 번에 명확해야 한다
- 선택한 입장은 강조 표시
- 시작 버튼은 선택 전 비활성화

---

### 3.3 토론 화면

핵심 게임 화면.

#### 목적

- AI와 유저의 턴을 집중해서 주고받게 한다

#### 주요 구성

- 상단 바
  - 주제 제목
  - 현재 라운드 표시 `1/3`, `2/3`, `3/3`
  - 종료 버튼 또는 뒤로가기 확인 버튼
- 대화 영역
  - AI 버블
  - User 버블
  - 최신 메시지 자동 스크롤
- 입력 영역
  - 텍스트 입력창
  - 글자수 `0/500`
  - 제출 버튼
- 보조 정보
  - 내 선택 입장 표시
  - AI는 반대 입장이라는 힌트

#### 상태

- AI 오프닝 표시
- 유저 입력 가능
- AI 응답 생성 중
- moderation 차단
- API 오류

#### UX 원칙

- 게임 진행 중에는 광고 노출 금지
- 입력 영역은 항상 고정
- AI 생성 중에는 skeleton 또는 thinking indicator 표시
- 모바일 키보드가 올라와도 입력창이 가려지지 않아야 한다
- 데스크톱에서는 대화 영역 폭을 제한해 너무 넓게 늘어나지 않게 한다

---

### 3.4 결과 화면

게임 종료 후 가장 중요한 전환 화면.

#### 목적

- 점수를 즉시 보여준다
- 저장/랭킹 등록/재도전 행동을 유도한다

#### 주요 구성

- 점수 카드
  - 총점
  - 짧은 총평
  - 점수 breakdown 요약
- 대표 문구 카드
  - 마지막 유저 대사
- CTA 그룹
  - `결과 저장하고 랭킹 올리기`
  - `한 판 더 하기`
  - `랭킹 보기`
- 닉네임 미설정 시
  - 저장 CTA 클릭 시 닉네임 모달 오픈
- 하단 배너/앵커 광고 영역

#### 상태

- 닉네임 미설정
- 이미 저장 완료
- retry ticket 있음
- retry ticket 없음
- 광고 로드 가능

---

### 3.5 닉네임 설정 모달

결과 저장 또는 랭킹 등록 시점에 등장하는 모달 또는 바텀시트.

#### 목적

- 현재 브라우저 세션에 닉네임을 연결한다

#### 주요 구성

- 제목
  - `랭킹에 표시할 닉네임을 정해주세요`
- 입력창
- 글자수 표시 `최대 10자`
- 확인 버튼
- 취소 버튼

#### UX 원칙

- 별도 회원가입처럼 보이지 않게 한다
- “이 브라우저에서만 기록이 저장됩니다” 안내를 짧게 표시

---

### 3.6 재도전 모달

무료 플레이 소진 후 `한 판 더 하기`를 눌렀을 때 등장.

#### 목적

- 광고 시청 후 추가 플레이 유도

#### 주요 구성

- 현재 상태
  - 무료 플레이 소진
  - retry ticket 보유 수
- CTA
  - `광고 보고 1회 더 하기`
  - `취소`
- 광고 실패 시 에러 문구

#### UX 원칙

- 강요 느낌보다 선택형 보상 구조로 보이게 한다
- 광고 길이는 고정 표기하지 않는다

---

### 3.7 랭킹 화면

일일 리더보드 화면.

#### 목적

- 경쟁 심리를 제공
- 저장 행동의 보상을 체감시킨다

#### 주요 구성

- 상단 헤더
  - `오늘의 랭킹`
  - 날짜
- 상위 3명 강조 카드
- 리스트형 순위
  - 순위
  - 닉네임
  - 점수
  - 대표 문구
- 내 순위 고정 카드
- 하단 배너/앵커 광고 영역

#### 상태

- 닉네임 미설정 사용자
  - 내 순위는 비노출 또는 `닉네임 설정 후 참여 가능`
- 데이터 없음
- 로딩

---

### 3.8 내 기록 화면

프로필/기록 화면.

#### 목적

- 저장한 게임 기록과 닉네임을 관리한다

#### 주요 구성

- 닉네임 카드
  - 현재 닉네임
  - 변경 버튼
- 요약 카드
  - 총 저장 게임 수
  - 최고 점수
- 기록 리스트
  - 날짜
  - 주제
  - 점수
  - 대표 문구
- 닉네임 미설정 상태 안내

#### 상태

- 닉네임 없음
- 저장 기록 없음
- 기록 있음

---

## 4. 반응형 레이아웃 규칙

### 4.1 모바일

- 기본 1열 레이아웃
- 하단 탭 고정
- 페이지 여백은 좁고 명확하게
- 리스트는 전체 세로 스크롤
- 토론 화면 입력창은 하단 고정

### 4.2 태블릿

- 홈/랭킹/내 기록은 넓은 카드 배치 허용
- 결과 화면은 점수 카드와 대표 문구 카드를 나란히 둘 수 있음
- 토론 화면은 집중도를 위해 여전히 단일 대화축 유지 권장

### 4.3 데스크톱

- 홈은 2열 가능
  - 왼쪽: 오늘의 주제와 CTA
  - 오른쪽: 상태 카드, 리더보드 요약
- 랭킹/내 기록은 넓은 리스트 또는 2단 카드 사용 가능
- 토론 화면은 중앙 정렬, 최대 폭 제한

---

## 5. 스크롤과 키보드 처리

### 5.1 토론 화면

- 메시지 영역만 스크롤
- 입력창은 항상 하단 고정
- 모바일 키보드 오픈 시 최신 메시지가 계속 보여야 함
- 입력창 높이는 일정 범위까지만 자동 확장

### 5.2 홈/랭킹/내 기록

- 페이지 전체 스크롤 허용
- 하단 탭과 광고 슬롯을 고려한 bottom padding 확보

### 5.3 모달/바텀시트

- 모바일은 바텀시트 우선
- 태블릿/데스크톱은 중앙 모달 허용
- 닉네임 모달은 키보드 오픈 시 입력 영역이 가려지지 않아야 함

---

## 6. 콘텐츠 길이 대응

### 6.1 오늘의 주제 카드

- 제목은 2줄까지 허용
- 옵션 A/B는 2줄까지 허용
- 그 이상은 말줄임 또는 카드 높이 확장

### 6.2 토론 버블

- 짧은 답변도 어색하지 않게 최소 높이 확보
- 긴 답변은 줄바꿈 우선, 강제 축약 지양

### 6.3 랭킹/기록 리스트

- 대표 문구는 2~3줄까지만 노출
- 나머지는 말줄임 처리 가능

---

## 7. 광고 슬롯 대응

### 배너/앵커

- 모바일: 하단 anchor 또는 콘텐츠 하단 배너
- 데스크톱: 콘텐츠 폭에 맞춘 인라인 배너
- 광고가 없을 때도 레이아웃이 무너지지 않아야 함

### rewarded 진입

- 광고 자체를 페이지의 일부처럼 설계하지 않는다
- `재도전 모달 -> 광고 호출 -> 복귀` 흐름으로만 취급한다

## 12. 디자인 방향 메모

- 홈은 “오늘의 주제”가 제일 먼저 보여야 한다.
- 게임 서비스이지만 너무 장난감처럼 보이지 않고, 가벼운 경쟁성과 논리 게임 느낌이 나야 한다.
- 탭 구조는 익숙하게 가되, 메인 CTA는 `홈`에 집중시킨다.
- 결과 화면은 성취감을 주는 카드형 구조가 좋다.
- 랭킹과 내 기록은 정보 밀도가 높아도 되지만, 홈과 토론 화면은 단순해야 한다.
- 토론 화면은 반응형이어도 항상 “집중형 읽기/쓰기 경험”이 우선이다.

## 13. 화면별 카피 문구 초안

### 13.1 홈

#### 기본 헤드라인

- 게임 이름: 내가 맞아, AI야!
- 메인 타이틀: `오늘의 주제`
- 서브 문구: `A와 B 중 하나를 고르고, AI와 3라운드 토론을 시작해보세요.`

#### 상태 카드

- 무료 가능: `오늘 무료 1회가 남아 있어요`
- 무료 소진: `오늘 무료 플레이는 이미 사용했어요`
- ticket 있음: `추가 도전 가능`
- ticket 없음: `광고 시청 후 한 판 더 도전할 수 있어요`
- 닉네임 없음: `랭킹 저장은 닉네임 설정 후 가능해요`
- 닉네임 있음: `현재 닉네임으로 기록을 저장할 수 있어요`

#### 버튼

- 메인 CTA: `토론 시작하기`
- 보조 CTA: `랭킹 보기`
- 내 기록 이동: `내 기록 보기`

#### 에러

- `오늘의 주제를 불러오지 못했어요. 잠시 후 다시 시도해 주세요.`

### 13.2 플레이 진입

#### 헤더

- 제목: `어느 쪽을 선택할래요?`
- 설명: `선택한 입장의 반대편에서 AI가 토론을 시작해요.`

#### 선택 카드

- 카드 상단: `선택지 A`
- 카드 상단: `선택지 B`

#### 버튼

- 선택 전: `입장을 선택해 주세요`
